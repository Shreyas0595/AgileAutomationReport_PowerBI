📊 Automation Report Dashboard
Overview
The Automation Report Dashboard provides a comprehensive view of automated testing results across software modules for a given sprint and release cycle. Built in Power BI, it enables teams to monitor test coverage, scenario execution, and pass/fail outcomes in a visually intuitive format.

Key Features:-
- Release & Sprint Filters  
- Dropdowns to select release name, sprint number, and date range for targeted reporting.

Module-Wise Test Case Distribution :-
- Bar chart showing the number of test cases executed per module:
- School Census (~580)
- PoST (~450)
- NI Assessment (~270)
- CTF (~270)
- DENI Return (~250)
- CES (~200)

Scenario Coverage :
- Line chart tracking scenario counts per module, ensuring breadth of test coverage.

Pass/Fail Status by Module :-
- Bar chart highlighting test outcomes:
- Example: School Census (278 PASS, 10 FAIL)
- Similar breakdowns for PoST, NI Assessment, DENI Return, and CTF.

Module Summary View :-
- Color-coded block chart listing all tested modules for quick navigation.

Sprint Summary Metrics:
- Total Sprints: 2
- Total Modules Tested: 5
- Total Test Cases: 1946
- Total Test Cases Passed: 1233
- Total Test Cases Failed: 32

Purpose:- 
- This dashboard helps QA teams and stakeholders:
- Track testing progress across modules.
- Identify modules with higher failure rates.
- Ensure adequate scenario coverage.
- Provide transparency in sprint-level automation outcomes.

Usage:-
- Open the Power BI dashboard.
- Select the desired Release Name, Sprint Number, and Date Range.
- Review module-specific charts and overall sprint summary.
- Use insights to prioritize bug fixes and improve test coverage.

Benefits:-
- Clear visibility into test execution status.
- Faster identification of problem areas.
- Data-driven decision-making for sprint planning.
- Improved collaboration between QA, Dev, and Product teams.
